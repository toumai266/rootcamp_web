# RootCamp PRD (Product Requirements Document)

## 1. 프로젝트 개요

### 1.1 프로젝트명
**RootCamp** - 정보보호 분야 종합 정보 포털

### 1.2 프로젝트 목적
정보보호 분야에 관심이 있는 사용자들에게 체계적이고 유용한 정보를 제공하는 정적 웹사이트 구축

### 1.3 참고 사이트
- [FMHY (freemediaheckyeah)](https://fmhy.net/) - 유사한 구조의 정적 웹사이트

### 1.4 기술 스택
- 정적 웹사이트 (Static Site)
- HTML, CSS, JavaScript
- 반응형 디자인

---

## 2. 핵심 기능 요구사항

### 2.1 메인 랜딩 페이지
**기능**: 정보보호 분야 월별 뉴스 요약

**상세 요구사항**:
- 월별로 정보보호 분야 주요 뉴스를 요약하여 제공
- 최신 뉴스가 상단에 표시
- 각 뉴스 항목은 제목, 요약, 출처, 날짜 포함
- 카테고리별 필터링 기능 (보안 취약점, 보안 사고, 정책/법규, 기술 동향 등)
- 검색 기능
- 월별 아카이브 기능

**우선순위**: 최우선 (P0)

---

### 2.2 정보보호 분야 진로 탐색

#### 2.2.1 진로별 공통 필요 기술 정보
**기능**: 정보보호 분야 모든 진로에서 공통적으로 필요한 기술 정보 제공

**상세 요구사항**:
- 공통 필수 기술 스택 목록
- 각 기술에 대한 설명 및 학습 가이드
- 학습 난이도 표시
- 추천 학습 순서
- 학습 자료 링크

**포함 기술 예시**:
- 네트워크 보안 기초
- 암호학 기초
- 시스템 보안
- 웹 보안 기초
- 리눅스/윈도우 시스템 관리
- 프로그래밍 언어 (Python, C/C++, JavaScript 등)
- 보안 도구 사용법

**우선순위**: 높음 (P1)

#### 2.2.2 진로별 특수 필요 기술 정보
**기능**: 각 진로별로 추가로 필요한 특수 기술 정보 제공

**상세 요구사항**:
- 진로 카테고리별 분류
- 각 진로에 대한 상세 설명
- 진로별 특수 기술 스택
- 진로별 학습 로드맵
- 진로별 취업 정보 및 전망

**포함 진로 예시**:
- **침투 테스트(Penetration Tester)**
  - 특수 기술: 침투 테스트 도구, 버그 바운티, 취약점 분석
- **보안 아키텍트(Security Architect)**
  - 특수 기술: 시스템 설계, 클라우드 보안, 아키텍처 패턴
- **보안 엔지니어(Security Engineer)**
  - 특수 기술: 보안 솔루션 구축, 인프라 보안, 자동화
- **보안 컨설턴트(Security Consultant)**
  - 특수 기술: 리스크 평가, 규정 준수, 보안 감사
- **보안 분석가(Security Analyst)**
  - 특수 기술: 로그 분석, 위협 인텔리전스, 사고 대응
- **악성코드 분석가(Malware Analyst)**
  - 특수 기술: 리버스 엔지니어링, 동적/정적 분석, 샌드박스
- **보안 연구원(Security Researcher)**
  - 특수 기술: 취약점 연구, 보안 연구 방법론, 논문 작성
- **CISO/보안 관리자**
  - 특수 기술: 보안 정책 수립, 팀 관리, 예산 관리

**우선순위**: 높음 (P1)

---

### 2.3 유용한 사이트 모음

#### 2.3.1 정보보호 분야 정부기관 사이트
**기능**: 정보보호 관련 정부기관 및 공공기관 사이트 모음

**상세 요구사항**:
- 기관명, 설명, 링크 제공
- 카테고리별 분류 (국가기관, 지자체, 공공기관 등)
- 각 사이트의 주요 기능 설명

**포함 사이트 예시**:
- 과학기술정보통신부
- 한국인터넷진흥원(KISA)
- 국가정보원
- 개인정보보호위원회
- 행정안전부
- 한국정보보호산업협회
- 국가사이버안전센터

**우선순위**: 중간 (P2)

#### 2.3.2 정보보호 분야 학습 사이트
**기능**: 정보보호 분야 학습에 유용한 온라인 교육 사이트 모음

**상세 요구사항**:
- 사이트명, 설명, 링크 제공
- 무료/유료 구분
- 난이도 표시 (초급/중급/고급)
- 학습 주제별 분류

**포함 사이트 예시**:
- OWASP
- PortSwigger Web Security Academy
- Hack The Box
- TryHackMe
- Cybrary
- SANS Cyber Aces
- 한국정보보호교육원
- 한국정보보호학회

**우선순위**: 높음 (P1)

#### 2.3.3 정보보호 분야 유용한 웹사이트
**기능**: 정보보호 분야에서 유용한 일반 웹사이트 모음

**상세 요구사항**:
- 사이트명, 설명, 링크 제공
- 카테고리별 분류 (도구, 자료, 커뮤니티 등)
- 각 사이트의 주요 기능 설명

**포함 사이트 예시**:
- CVE (Common Vulnerabilities and Exposures)
- Exploit-DB
- SecurityFocus
- Packet Storm Security
- GitHub Security Advisories
- 보안 뉴스 사이트들
- 취약점 데이터베이스

**우선순위**: 중간 (P2)

#### 2.3.4 정보보호 분야 웹진/블로그 사이트
**기능**: 정보보호 분야 전문 웹진 및 블로그 모음

**상세 요구사항**:
- 사이트명, 설명, 링크 제공
- 국내/해외 구분
- 주제별 분류
- 업데이트 빈도 표시

**포함 사이트 예시**:
- 보안뉴스
- 보안공지
- Dark Reading
- Krebs on Security
- The Hacker News
- Schneier on Security
- 개인 보안 전문가 블로그들

**우선순위**: 중간 (P2)

## 3. 비기능 요구사항

### 3.1 성능
- 페이지 로딩 시간: 3초 이내
- 정적 사이트로 빠른 응답 속도
- 이미지 최적화

### 3.2 사용성
- 직관적인 네비게이션
- 반응형 디자인 (모바일, 태블릿, 데스크톱)
- 검색 기능
- 카테고리별 필터링

### 3.3 접근성
- WCAG 2.1 AA 수준 준수
- 키보드 네비게이션 지원
- 스크린 리더 호환

### 3.4 디자인
- 깔끔하고 모던한 UI/UX
- FMHY와 유사한 카드 기반 레이아웃
- 다크 모드 지원 (선택사항)

### 3.5 유지보수성
- 정적 사이트 생성기 사용 고려 (Jekyll, Hugo, Next.js 등)
- 마크다운 기반 콘텐츠 관리
- 버전 관리 (Git)

- 진로 상세 페이지 구현
